## Branch Strategy

- `main`: 안정화된 최종 코드만 병합합니다.
- `feature/<name>`: 기능 개발 브랜치입니다.
- `fix/<name>`: 버그 수정 브랜치입니다.
- `docs/<name>`: 문서 수정 브랜치입니다.

## Collaboration Rules

1. 작업 전 최신 `main`을 가져옵니다.
2. 각자 기능별 브랜치를 만들어 작업합니다.
3. 커밋 메시지는 변경 의도가 드러나게 작성합니다.
4. `main`에는 직접 푸시하지 않고 Pull Request로 병합합니다.
5. Pull Request는 최소 1명 이상 리뷰 후 병합합니다.
